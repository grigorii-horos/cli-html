# Abbreviations

Abbreviations allow you to define expansions for acronyms and short forms.

## Technical Abbreviations

The HTML specification is maintained by the W3C.

You can write CSS or use SCSS for styling.

API endpoints return JSON or XML data.

*[HTML]: Hyper Text Markup Language
*[W3C]: World Wide Web Consortium
*[CSS]: Cascading Style Sheets
*[SCSS]: Sassy CSS
*[API]: Application Programming Interface
*[JSON]: JavaScript Object Notation
*[XML]: Extensible Markup Language

## Programming Abbreviations

Use OOP principles with SOLID design patterns.

Write DRY code and avoid WET solutions.

Follow TDD or BDD for testing.

*[OOP]: Object-Oriented Programming
*[SOLID]: Single Responsibility, Open-Closed, Liskov Substitution, Interface Segregation, Dependency Inversion
*[DRY]: Don't Repeat Yourself
*[WET]: Write Everything Twice
*[TDD]: Test-Driven Development
*[BDD]: Behavior-Driven Development

## Web Development

Build SPAs with React, Vue, or Angular.

Use REST or GraphQL APIs.

Deploy with CI/CD pipelines.

*[SPA]: Single Page Application
*[REST]: Representational State Transfer
*[GraphQL]: Graph Query Language
*[CI/CD]: Continuous Integration/Continuous Deployment

## Protocols and Standards

Access websites via HTTP or HTTPS.

Use SSH for secure connections.

Transfer files with FTP or SFTP.

*[HTTP]: Hypertext Transfer Protocol
*[HTTPS]: Hypertext Transfer Protocol Secure
*[SSH]: Secure Shell
*[FTP]: File Transfer Protocol
*[SFTP]: Secure File Transfer Protocol

## Database Terms

Use SQL for RDBMS queries.

Try NoSQL for unstructured data.

Implement CRUD operations.

*[SQL]: Structured Query Language
*[RDBMS]: Relational Database Management System
*[NoSQL]: Not Only SQL
*[CRUD]: Create, Read, Update, Delete

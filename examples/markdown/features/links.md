# Links

Different ways to create links in Markdown.

## Inline Links

This is [an inline link](https://example.com).

This is [a link with a title](https://example.com "Link Title").

## Reference Links

This is [a reference link][ref1].

This is [another reference link][ref2].

You can also use [implicit references].

[ref1]: https://example.com
[ref2]: https://example.com/page "Optional Title"
[implicit references]: https://example.com

## Automatic Links

<https://example.com>

<user@example.com>

## URL Auto-linking

GitHub Flavored Markdown automatically links URLs:

https://github.com

www.example.com

## Links with Formatting

This is a **[bold link](https://example.com)**.

This is an *[italic link](https://example.com)*.

This is a `[code link](https://example.com)`.

## Links in Lists

- [First link](https://example.com)
- [Second link](https://example.com)
- [Third link](https://example.com)

## Multiple Links in Text

You can visit [GitHub](https://github.com) for repositories, [Stack Overflow](https://stackoverflow.com) for questions, or [MDN](https://developer.mozilla.org) for documentation.

## Internal Links

Link to [another section](#automatic-links) in the same document.

Link to [the top](#links) of this page.
